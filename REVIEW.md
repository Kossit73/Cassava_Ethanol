# Cassava Bioethanol Model – Financial Modelling Recommendations

## 1. Separate real growth from inflation multipliers
The landing-page helper already lets users describe year-on-year percentage growth for each direct cost item, and the helper compounds those growth rates into the monthly rows that are written back to the table. However, the cost schedule then multiplies the entire direct-cost matrix by the CPI factors derived from the inflation schedule, meaning a user-entered 5% yearly increase is automatically compounded again by the CPI rate for the same year. This double counts inflation and overstates COGS unless the intent is to work in real (inflation-free) terms. Consider either (a) clarifying in the UI that yearly increments are real values and therefore combined with CPI, or (b) tagging each cost item as "inflation-indexed" so only items that do not already contain nominal growth receive the CPI uplift during `_prepare`.【F:streamlit_app.py†L1472-L1567】【F:bioethanol_model/schedules.py†L600-L639】

## 2. Preserve user shape of monthly profiles
When the helper rewrites a cost item it applies one value per calendar year to every month in that year. This makes seasonal patterns or mid-year adjustments impossible even if the user entered richer data before opening the helper. Consider letting the form either import the existing monthly series as defaults or allowing secondary breakpoints (e.g., quarterly or effective month columns) so that not every month in a year is forced to the same number after the helper runs. That would stop the helper from flattening costs such as harvest campaigns or insurance premia that are paid in one month but currently get spread evenly across twelve months.【F:streamlit_app.py†L1531-L1570】

## 3. Reconcile scenario overrides with edited direct-cost tables
The scenario preparation step rewrites cassava feedstock rows by replacing their `Amount` with `tons × cost per ton`. If a user has already run the yearly helper (perhaps in absolute mode to capture negotiated supply rates), those curated amounts are discarded every time the model is rebuilt for a different scenario. You could move the scenario-specific adjustment upstream into the helper (e.g., treat the farm/purchase mix as a driver for the helper baseline) or persist the user-entered nominal values and only adjust rows that still match the untouched template. This would keep scenario toggles from silently overriding deliberate edits.【F:bioethanol_model/financial_model.py†L161-L195】

## 4. Align inflation factors with the planning start
`_build_inflation_factors` anchors the first projection year at a multiplier of 1.0 and then compounds subsequent CPI rates. When the planning start is later than the overall start year, months before the first CPI entry still receive the default 1.0 multiplier even if the planning horizon should begin mid-series. Reviewing whether the inflation schedule should be aligned to the planning start (or default to the first CPI year present in the table) would prevent mismatches between the target start month and the inflation curve applied to the cost schedules.【F:bioethanol_model/schedules.py†L590-L636】
